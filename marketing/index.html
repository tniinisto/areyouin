Easy event handling

It all started from a group of friends playing beach volleyball and the fact that setting the game times that would suite everyone was sometimes difficult and time consuming! There were multiple mails, messages and phone calls to set the time that would suite. During this came the idea that an application could help with this. The first version came out around summer 2012 and after that we have been using it. During the years it has developed further and features that have been seen usefull were added. The app's main principal has been on the ease of usage.

It is a responsive web application that works basically on all modern web browsers on PCs, tablets and smartphones. Requires no installations and new features are available for everyone instantly. The main functionalities are events and participants who can easily set their status IN or OUT. Event has start & end times, location and of course the participants in it. There is also a chat available for team's members where it is easy to get the message for everyone. On top of these there are also things like the user's profile and activity charts.

features

Events have location, start & end times and participants with it. Summary on the event participation is shown at the top of each event, where you can see how many is going to participate and how many have been invited for the event. Clicking the summary maximises the event view and each participant's status is shown separately. User's own IN/OUT status is always visible and directly accessible on the event. Clicking the event's location opens Google Maps in a new tab on browser showing the event's location. Each user's "avatar" is rounded with a yeallow border in the event indicating that the user has seen the event. Web notifications are also available same as with chat, notifications are given on event changes.

Chat is a basic character based messaging board. It shows all messages within the group. On new messages an indication is shown on the app navigation. On PC & tables browsers also a notification with sound is given if the user is logged in and even if the browser is minimized. Nowadays Chrome & Firefox support web notifications which gives a notification message on the operating system. IE, Edge & Safari browsers give only a sound when a notifications occurs. Smartphones doen't yet support web notifications.

Profile shows user's basic information like Nickname, First & Last names, email (This is also the userID used in login), phone and wether mail notifications are wanted on new events. There is also the password change and own "avatar" upload features.

Admin user has the ability to set and update events, add new users to the group and manage locations. On locations can also be decided if weather info wanted to be displayed on the events page for the location. He/She can also promote other users as admins.
